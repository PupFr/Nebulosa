Data Retention and Protection Policy

NEBULOSA BOT - Zoom Meeting Management Integration

Last Updated: July 23, 2025

1. Data Retention Overview

NEBULOSA BOT follows a minimal data retention approach:

  • No persistent data storage implemented
  • All data processing occurs in memory only
  • Data is automatically purged after each session
  • No long-term data retention requirements

2. OAuth Token Handling

  • OAuth tokens are never stored persistently
  • Tokens are processed in memory during active sessions
  • Automatic expiration follows Zoom's token lifecycle
  • No token refresh or long-term storage

3. Meeting Data Processing

  • Meeting information accessed only during active operations
  • No meeting data cached or stored locally
  • Real-time processing without data persistence
  • Data retrieved fresh from Zoom APIs when needed

4. Data Protection Measures

  • All data encrypted in transit via HTTPS/TLS 1.2+
  • Memory-only processing reduces exposure risk
  • No data backup or archival processes
  • Secure credential management using environment variables

5. User Data Rights

  • No data deletion requests needed (none stored)
  • Users can revoke access immediately via Zoom settings
  • Complete data portability (no data to export)
  • Instant access revocation capabilities

6. Compliance

  • Minimal data collection principle followed
  • No cross-border data transfer issues
  • Stateless architecture ensures data minimization
  • Regular security reviews and updates
